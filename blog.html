Answer: The scope of variables depends on whether they are declared using var, let, or const.
  • var: Function-scoped, and variables are hoisted.
  • let: Block-scoped, and variables are not hoisted.
  • const: Block-scoped, similar to let, but cannot be reassigned.
Keyword Scope Update/Re-declare Initialization
var Functional Scope Can be updated and re-declared Can be declared without initialization
let Block Scope Can be updated but cannot be re-declared Can be declared without initialization
const Block Scope Cannot be updated or re-declared Must be initialized at the time of declaration

Answer: Null and undefined are special values in JavaScript used to represent the absence of a value.
  • null: Intentional absence of any object value.
  • undefined: Default value of variables that are not initialized.

Comparison of Undefined and Null

Property Undefined Null
Meaning Variable declared but not assigned a value Assignment value representing no value
